# AGENTS.md 構成リファレンス

## 基本テンプレート

```md
# AGENTS.md

## 基本方針

- 回答は日本語で行う。
- 作業前に既存コードと設定ファイルを確認する。
- 変更後は関連する検証コマンドを実行する。
- ユーザーが作った変更を勝手に戻さない。

## プロジェクト概要

- このリポジトリは <目的> のためのプロジェクトである。

## 技術スタック

- 言語 / ランタイム: <確認できたもの>
- 主要フレームワーク / ライブラリ: <確認できたもの>
- データ保存 / 外部サービス: <確認できたもの>
- ツールチェーン: <確認できたもの>

## 開発コマンド

- 依存関係のインストール: `<実在するコマンド>`
- 開発サーバー: `<実在するコマンド>`
- ビルド: `<実在するコマンド>`
- テスト: `<実在するコマンド>`
- lint / format: `<実在するコマンド>`

## 詳細情報の場所

- ビルド手順: `<path>`
- テスト方針: `<path>`
- アーキテクチャ: `<path>`
- ドメイン知識 / スキーマ: `<path>`

## ディレクトリ構成

- `<path>`: <責務>
- `<path>`: <責務>

## 環境変数と秘密情報

- 秘密情報をログや回答に出力しない。
- 秘密情報を含みうるファイルは値を表示せず、キー名、存在、用途だけ確認する。

## Git 運用

- 既存のユーザー変更を勝手に戻さない。
- commit ルールや hook がある場合は尊重する。

## 禁止事項

- 実在しないコマンドを書かない。
- README と同じ説明を重複させない。
- 仕様書やロードマップを全文貼り付けない。
- 細かいコードスタイル規則を自然言語で大量に書かない。
- 限定的な作業にしか関係しない指示をルートの AGENTS.md に置かない。
```

## 情報源の扱い

特定のツールやサービスを前提にしない。次のように、リポジトリやユーザー入力から確認できたものだけ扱う。

- パッケージ管理: lockfile、manifest、README、CI から判断する。
- タスク実行: scripts、Makefile、task runner、CI から判断する。
- 外部ドキュメント: ユーザーが URL、ファイル、取得方法を渡した場合だけ読む。
- デプロイ先: 設定ファイルや CI から確認できた場合だけ書く。
- ローカル環境管理: 専用設定ファイルが存在し、実際に使われている場合だけ書く。

## 段階的開示

AGENTS.md にすべてを書かず、必要なときに読む文書へ誘導する。

- ルートの AGENTS.md: ほぼすべての作業に効く最小限のルールを書く。
- 詳細文書: ビルド、テスト、設計、スキーマ、運用、ドメイン知識など、タスク依存の情報を書く。
- 参照はコピーより優先する。詳細文書や実装ファイルへのパスを示し、同じ情報を重複させない。
- コード例を貼るより、信頼できる実装ファイルや設定ファイルへの参照を優先する。

## 判断基準

AGENTS.md に残すか迷ったら、次の問いで判断する。

- AI コーディングエージェントが毎回の作業で守る必要があるか。
- その指示がないと破壊的変更や誤った検証につながるか。
- README より AGENTS.md にあるほうが実用的か。
- 現在のリポジトリから確認できる事実か。
- 機械的に検出または修正できる内容を、自然言語指示として重複させていないか。

すべてに当てはまらない内容は、README、`docs/architecture.md`、`docs/roadmap.md` などに分ける。
